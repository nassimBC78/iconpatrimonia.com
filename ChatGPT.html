🚀 1. Utiliser GitHub Pages (idéal pour sites vitrines, portfolios, landing pages)

  • 💡 Avantages : 100% gratuit, sans pub, rapide, fiable, hébergé par GitHub (Microsoft).

  • ⚠️ Limites : statique uniquement (HTML/CSS/JS), pas de base de données.

  • 📌 Pour qui : sites vitrine, CV, documentations, portefolios.

🔧 2. Utiliser Netlify ou Vercel

  • 💡 Avantages : gratuit, très pro, déploiement automatique depuis GitHub, supporte frameworks modernes (Next.js, Nuxt, etc).

  • ⚠️ Limites : usage raisonnable (mais suffisant pour 95% des petits sites).

  • 📌 Pour qui : devs, startups, landing pages, blogs statiques.

🌐 3. Utiliser un constructeur de site gratuit (genre Wix, Webflow, Carrd, etc.)

  • 💡 Avantages : no-code, rapide, intuitif.

  • ⚠️ Limites : nom de domaine avec leur sous-domaine (monsite.wixsite.com), fonctionnalités bloquées sans abonnement, pub possible.

  • 📌 Pour qui : petites boîtes, indépendants, landing page rapide.

🧱 4. Utiliser WordPress.com (plan gratuit)

  • 💡 Avantages : simple, puissant, SEO-friendly.

  • ⚠️ Limites : sous-domaine (.wordpress.com), peu de personnalisation sans payer.

  • 📌 Pour qui : blog, petit site de présentation.

💡 Et pour le nom de domaine ?

  • Gratuit : tu restes avec le sous-domaine (ex: monprojet.github.io ou monsite.wixsite.com)

  • Pro : tu achètes un domaine personnalisé (≈10€/an) et tu le connectes.

✅ Conclusion : Le combo gagnant

Si tu veux quelque chose de pro, sans pub, rapide, alors :

  • GitHub Pages + nom de domaine ≈ 0€ (ou juste 10€/an pour ton domaine)

  • Ou Netlify si tu veux un peu plus de flexibilité technique.

2. Achète ton domaine

Tu peux acheter chez n’importe quel registrar (vendeur de domaines), par exemple :

RegistrarAvantages
NamecheapTrès bon support, facile à configurer, pas cher
OVH🇫🇷, solide, bien si tu veux tout héberger chez eux aussi
Gandi🇫🇷 aussi, éthique, fiable
IONOSTrès grand public, souvent des promos

👉 Compte entre 8€ et 15€/an pour un .fr ou .com. Tu paies avec CB et tu as accès à ton interface pour gérer le domaine.

3. Connecte ton domaine à ton site web

Quand ton site est prêt (sur GitHub Pages ou autre), tu dois pointer ton domaine vers ton hébergement :

  • Tu ajoutes un CNAME ou A record pour que www.tonsite.fr affiche ton site.
